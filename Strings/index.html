Strings

Javascript "char" is a String. There's no Character datatype in JS. Also, String can be denoted by Single Quote ('') or Double Quote ("") in JS.

Strings are immutable. You cannot change a character within a string with something like var myString = "abbdef"; myString[2] = 'c'. The string manipulation methods such as trim, slice return new strings.

In the same way, if you have two references to the same string, modifying one doesn't affect the other

So all that said, Strings are assigned by reference, not by value.

You may use single or double Quotes for strings but as a standard, developers prefer using one notation tending towards use of Double Quotes

Operations

Concatenation

var a = 'a'; var b = 'b'; var c = a + b;

Comparison

var a = 'a'; var b = 'b'; if (a < b) { // true console.log(a + ' is less than ' + b); } else if (a > b) { console.log(a + ' is greater than ' + b); } else { console.log(a + ' and ' + b + ' are equal.'); }

Escaping strings

Sometimes, your code will include strings which are very long. Rather than having lines that go on endlessly, or wrap at the whim of your editor, you may wish to specifically break the string into multiple lines in the source code without affecting the actual string contents. There are two ways you can do this.

You can use the + operator to append multiple strings together, like below using String Concat: var longString = "This is a very long string which needs " + "to wrap across multiple lines because " + "otherwise my code is unreadable.";

Or you could do this: var longString = "This is a very long string which needs \ to wrap across multiple lines because \ otherwise my code is unreadable.";

Templating

Starting with ECMAScript 2015, string literals can also be so-called Template literals: `hello world` `hello! world!` `hello ${who}` tag `${who}` What do you use it for? var me = { firstName: "Maya", lastName: "Shavin", } function saysHi (obj) { return `Hi there, I'm ${obj.firstName} ${obj.lastName}`; } console.log(saysHi(me));
