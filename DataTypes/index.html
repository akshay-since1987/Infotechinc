Datatypes

JavaScript has dynamic types. This means that the same variable can be used to hold different data types.

Objects

var person = { firstName:"John", lastName:"Doe", age:50, eyeColor:"blue" };

Number

In Javascript, Any type of "Number" is just a number. There's no Integer, Decimal, Float, BigInt, SmallInt, etc in Javascript. var x = 10; var y = 0.2; var z = Infinity; var a = 2 / 0; // == Infinity

Strings

Javascript "char" is a String. There's no Character datatype in JS. Also, String can be denoted by Single Quote ('') or Double Quote ("") in JS.

Strings are immutable. You cannot change a character within a string with something like var myString = "abbdef"; myString[2] = 'c'. The string manipulation methods such as trim, slice return new strings.

In the same way, if you have two references to the same string, modifying one doesn't affect the other

So all that said, Strings are assigned by reference, not by value.

You may use single or double Quotes for strings but as a standard, developers prefer using one notation tending towards use of Double Quotes var a = 'char'; // "char" is a String var b = 'c'; // "c" is a String var c = "String"; // "String" is a String var d = "S"; // "S" is a String var e = 's' + "tring"; // "string" is a String var f = ['s', 't', 'r',"i",'n','g'].join(); // "string" is a String

Bool or Boolean

As always boolean can contain only 2 values true or false.

Word of Caution: Do not confuse with "truthy" (Indicates that this variable will always have value even if the value is "0" or "undefined" or [] or "false") and "falsey" (Indicates that the variable maybe null or undefined) values that developers talk about. Truthy/Falsey are the values that are refered and not the type. In order to avoid confusion during coding always use double inversion as (!!somevariable). This will ;ensure you always get true or false and not "truthy/falsey" values.

Note in examples below that As long as the variable returns a value that is not undefined or null or false or 0, Double inversion causes it to return true.

var x = false; // false var y = true; // true var somevar = 0; // 0 is a number not boolean var z = !!somevar; // Double inversion of 0 results in false var a = !!true; // Double inversion of true is true var b = !!undefined; // Double inversion of string undefined is true var c = !!"string"; // Double inversion of any string is true var d = !!"false"; // Double inversion of false as a string is also true var e = !![]; // Double inversion of empty array is true var f = !!['s','t','r', 'i', 'n', 'g']; // Double inversion of array with values is true var g = !!null; // Double inversion of null is false var h = !!undefined; // Double inversion of undefined is false var i = !!{}; // Double inversion of Object is true

Undefined

In Javascript, we have a special type Undefined, unlike many other languages. A variable when declared is always set with undefined value (This will be explained later in sessions covering Scope and Variable hoisting).

var x; // x is undefined var y = undefined // y is undefined var z = "undefined" // z is a string NOT undefined

Typeof Operator

typeof Operator in JS is used to determine the type of object typeof x; // x is undefined typeof "string"; // "string" is a string typeof true; // true is a boolean typeof 0.002; // 0.002 is a number typeof ["a"]; // ["a"] is an array typeof {}; // {} is an object
